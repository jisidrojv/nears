# NEARS Template - README EN

NEARS is an SSR web framework in Rust inspired by the ergonomics of Next.js, but implemented with a native Rust stack: Axum, Maud, PostgreSQL, HTMX, Alpine, and Tailwind CLI. The app renders HTML on the server, uses file-based routing, separates `page.rs` and `route.rs`, and keeps authentication and data access on the server side.

## Stack

- `axum` + `tokio`
- `maud`
- `sqlx` + PostgreSQL
- `htmx`
- `alpine.js`
- Tailwind CSS v4 CLI
- JWT + `CookieJar` + `bcrypt`

## Base architecture

```text
nears/
├── app/            # Visual routes and HTTP handlers
├── components/     # Global components
├── lib/            # Auth, DB, DAL, and types
├── migrations/     # SQL schema files
├── public/         # Static assets
├── scripts/        # Seed and utilities
├── build.rs        # Router generator
├── main.rs         # Axum server
├── Makefile
└── nears           # Local CLI
```

## Principles

1. Server-rendered HTML.
2. File-based routing generated by `build.rs`.
3. `page.rs` for views and `route.rs` for HTTP handlers.
4. DAL-style auth guards on the server.
5. `HX-Trigger` when there is no full redirect, ephemeral cookie only when there is a full redirect.
6. Sensitive responses use `no-store`.

## Requirements

Before running the project on your machine, you need:

1. Stable Rust
2. Cargo
3. PostgreSQL running locally
4. Tailwind CSS standalone CLI
5. Linux for `tailwindcss-linux-x64`, or the matching Tailwind standalone binary for your OS

## Local setup

### 1. Clone the repository

```bash
git clone https://github.com/jisidrojv/nears.git
cd nears
```

### 2. Download Tailwind CSS standalone CLI

Linux:

```bash
curl -sLO https://github.com/tailwindlabs/tailwindcss/releases/latest/download/tailwindcss-linux-x64
chmod +x tailwindcss-linux-x64
```

If you are on macOS or Windows, download the matching standalone binary from the Tailwind CSS releases page and place it in the project root as `tailwindcss-linux-x64`, or adjust the `TAILWIND` variable in `Makefile`.

### 3. Create the database

Create a PostgreSQL database, for example:

```sql
CREATE DATABASE nears;
```

### 4. Configure `.env`

Create a `.env` file inside `nears/`:

```env
HOST=127.0.0.1
PORT=3000
DATABASE_URL=postgresql://user:password@localhost:5432/nears
JWT_SECRET=replace-this-with-a-long-random-secret
```

You can also start from:

```bash
cp .env.example .env
```

### 5. Reset schema and seed initial data

```bash
./nears reset
```

This command:

1. recreates the schema
2. runs migrations
3. executes `scripts/seed.rs`

## Run in development

```bash
./nears dev
```

That compiles CSS and starts the server.

Open:

```text
http://127.0.0.1:3000
```

## Default credentials

After seeding, the app creates this user:

- email: `admin@nears.dev`
- password: `nears/admin!`

## Useful commands

- `./nears dev` -> compile CSS and start the server
- `./nears reset` -> recreate DB and run the seed
- `./nears build` -> production build
- `./nears clean` -> clean artifacts

## Authentication pattern

- private pages: `require_session(&jar)`
- private HTMX endpoints: `require_htmx_session(&jar)`
- sensitive responses: `no_store_response(...)`
- never write manual `Set-Cookie` headers for session or toast

## Production build

```bash
./nears build
```

The binary is generated at:

```text
target/release/nears-template
```

You also need to copy:

1. `public/`
2. `.env`
